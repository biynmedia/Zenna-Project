Installation de Symfony 3.4

https://symfony.com/doc/3.4/index.html
Suivi des étapes de setup : https://symfony.com/doc/3.4/setup.html

Pré-requis

  • Version PHP : 7.1 (recommandé), 5.5 minimum
  • Extensions PHP : xdebug, pdo_sqlite, intl, openssl
  • IDE : PHPStorm (recommandé), Netbeans ou autre IDE
  • Pour les utilisateurs Windows : cmder

Installation simple grâce au Symfony Installer :

Symfony fournit une application dédiée Symfony Installer qui nous permet de mettre en place / installer rapidement notre projet.

Mise en place du Symfony Installer

Dans votre terminal windows à la racine de votre projet, par exemple C:\wamp64\www\symfony :

php -r "file_put_contents('symfony', file_get_contents('https://symfony.com/installer'));"

Cette installation ne se fait qu’un seule fois pour votre projet.

Si tous s’est bien passé, vous devriez voir un fichier symfony dans votre dossier.
Pour vérifier le fonctionnement de votre Symfony Installer vous pouvez lancer la commande suivante :

php symfony

Création du Projet

La création de notre projet est ensuite simplifié.
Toujours dans notre terminal :

php symfony new zenna 3.4

Si tous s’est bien passé, vous devriez voir dans votre terminal :

OK  Symfony 3.4.8 was successfully installed

Composer

Vous pouvez également utiliser composer pour créer votre projet.

Doc de Référence :
https://symfony.com/doc/3.4/setup.html#creating-symfony-applications-with-composer

Composer est un gestionnaire de dépendance qui vous permettra de définir les différentes dépendances pour votre projet. Composer utilise un fichier composer.json qui contient plusieurs informations sur le projet dont la liste des librairies utilisées. Il est ensuite capable de télécharger automatiquement ces librairies (et les dépendances associées) et de générer un autoloader pour les utiliser simplement dans vos projets PHP.

Packagist

Packagist est le repository principal de Composer. Il intègre des packages public installable à partir de Composer.

SSL certificate problem

Si vous avez une erreur du type :

[GuzzleHttp\Exception\RequestException]
cURL error 60: SSL certificate problem: unable to get local issuer certificate

Les certificats ne sont pas correctement installé sur votre système.

  1. Créez un dossier certs dans votre serveur web local. Exemple : C:\wamp64\certs
  2. Téléchargez la liste des derniers certificats : https://curl.haxx.se/ca/cacert.pem
  3. Déplacer le fichier téléchargé cacert.pem dans votre dossier certs
  4. Mettez à jour votre fichier php.ini

Pour mettre à jour votre fichier php.ini vous devez repérer votre version de PHP.
Dans votre terminal php -v, ensuite dans votre dossier php correspondant, localisez votre fichier php.ini puis modifier la ligne suivante :

curl.cainfo = "C:\path\to\cacert.pem"

Par Exemple :

curl.cainfo = "C:\wamp64\certs\cacert.pem"

Lancement du Serveur

Vous avez dû remarquer, Symfony viens avec un serveur PHP embarqué.
Vous pouvez l’utiliser directement si vous le souhaitez en lieu et place de votre serveur WAMP.

 cd zenna
 php bin/console server:run

Ouverture dans le navigateur

Symfony Installer à installé pour nous automatiquement toute l’architecture de Symfony.
Nous n’avons plus qu’à nous rendre sur notre URL de développement :

http://localhost:8000/

Barre de Débogage

Observez la barre de débogage. Au fur et à mesure que nous installerons de nouveaux packages, nous verrons de nouvelles options apparaître.

Petit tour de découverte des options de la barre.

Installation et Configuration du Plugin Symfony pour PHPStorm

Le plugin symfony pour PHPStorm permettra à notre IDE de reconnaître toutes les fonctionnalités de Symfony.

Nous allons installer :

  1. PHP Annotations
  2. PHP Toolbox
  3. Symfony Plugin

Nous pouvons ensuite activer le plugin et redémarrer PHPStorm.
PS : Vérifiez que App Directory pointe sur le dossier app et WebDirectory sur le dossier web
